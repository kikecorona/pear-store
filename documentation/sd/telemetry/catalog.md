## Details

The core metrics emitted by the catalog service include event logs, system metrics, and request traces.

Event logs from the catalog service can be used for monitoring interactions between the review service and the catalog service [S4](../services/review.md), as well as interactions with other services via the catalog API [S5](../services/catalog.md). The catalog service generates event logs related to these interactions [S1](../services/catalog.md).

System metrics from the catalog service provide insights into its performance, such as CPU usage or memory consumption [S2](README.md) (further detail TBD). These metrics can be used by the Search service to monitor its execution time, throughput, and latency.

Request traces from the catalog service help identify bottlenecks in cart management processes [S3](../services/search.md). The catalog service owns the catalog database and is responsible for managing user carts, which implies that request traces related to cart management are generated by the catalog service [S7](../services/catalog.md).

Note that the specific data stored in the cart database for each user is unclear (further detail TBD) [S4](../services/review.md), but the Cart service stores information about items a user has added to their cart, including `user_id`, `items`, and other relevant data [S9].
